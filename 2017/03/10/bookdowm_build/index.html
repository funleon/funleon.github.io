bookdown網站建立初探

March 10, 2017 架站

偶然看到金門大學陳鍾誠教授發表的bookdown，是以Markdown撰寫書籍的平台，剛好在公司內部要嘗試將技術文件Markdown化來發布，但不方便放到gitbook上，因此就嘗試架陳教授的bookdown來試試，這裡將建置和使用過程做個備忘。

  1. 依據README.md

    • $ git clone https://github.com/ccckmit/bookdown.git
    • $ cd bookdown
    • $ npm install
      install發生錯誤，如下圖，原因是/package.json第17行”mz”: “^2.6.0”之後備註文字無法解析，刪除後即可進行。

    • node server:在Mac上執行權限不足，加上sudo即可，run起來在http://localhost 可瀏覽，https亦可。

  2. 在bookdown上建立書籍

    • 先在/setting/setting.mdo新增帳號密碼。
    • 以system(寫在setting.mdo中)帳密登入，在Top menu點擊”使用者”–>”系統”，左側menu點擊使用者，再按下Top menu點擊”使用者”–>”編輯”，在畫面表格新增一筆帳號profile。
    • 以新增的帳號登入，Top menu有”寫書”功能，但輸入書名submit後會自動建立兩個篇文檔目錄，但還沒有實體文檔產生。
    • 要建立實體md文檔，就點擊左側目錄如README，點擊Top menu”使用者”–>”編輯”，以Markdown格式編輯內容，然後點擊Top menu”使用者”–>”存檔”，文檔實體路徑在”/data/book/新書名”之下。
    • 若要新增章節，可以按左上角”內容”，會顯示書籍結構(實體路徑/data/book/新書名/book.mdo)。

  3. 想法

    • 依照上述步驟即可建置完成一個以Markdown書籍為基礎的網站，我目前嘗試運用於公司內部技術文件之分享，因為比較是單向發佈，所以我把寫書、編輯、登出登入功能都從Menu上拿掉。
    • 目前感覺比較大的問題是網站的UX不太好，操作動線不順，同類型功能分散，如寫書和編輯、存檔隔得很遠，雖然理解可能是考慮編輯和存檔功能不只用在寫書，包含系統管理都在用，但對一般寫書的使用者來說，並不直覺，寫一本書下來滑鼠要移來移去…手酸，用筆電的觸控板就更累了。
    • 另外還有系統管理的部分，使用者的profile設定似乎沒有作用，還是需要到setting.mdo把home path改掉，Sign in建立帳號的功能也不能用(也許連上DB就可以吧，還沒測)。
    • 距離產品化感覺還有蠻大一段gap要跨越，也可能是陳教授有產品版沒公開也未可知，之後功能有較完整或自己有時間和能力來改改，還是會考慮作為公司內部IT部門共享平台來用。
